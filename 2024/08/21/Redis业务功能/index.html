4、推送

4.1、Feed流常见模式

（1）TimeLine

  • 优点：信息全面，不会有缺失，且实现简单。
  • 缺点：噪音多，用户不一定感兴趣，内容获取率低。
  • 实现方案：
    • 拉模式（读扩散）：不推荐
    • 推模式（写扩散）：不推荐
    • 推拉结合：推荐

（2）智能排序

  • 优点：投喂用户感兴趣的信息，用户黏度高，易沉迷。
  • 缺点：若 算法不精准，可能起到相反作用。

4.2、收件箱

  • 使用SortedSet结构来存储。
  • Feed流滚动分页：ZSet：zrevrangebyscore … withscores limit …
    • max / min / offset / count
      • max：当前时间戳 | 上一次查询的最小时间戳
      • min：0
      • offset：0 | 上次结果中最小值相同的个数
      • count：3
